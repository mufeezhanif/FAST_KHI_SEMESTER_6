# Submission Checklist

Use this before final submission.

## Code Completeness
- [ ] Preprocessing implemented: tokenization, lowercasing, stopwords, stemming.
- [ ] Inverted index built correctly.
- [ ] Positional index built correctly.
- [ ] Indexes saved to disk and loaded back.
- [ ] Simple Boolean queries work.
- [ ] Complex Boolean queries with parentheses work.
- [ ] Proximity `/k` queries work.

## GUI Completeness (Bonus)
- [ ] Streamlit app launches without errors.
- [ ] Query type selector implemented.
- [ ] Clear and friendly layout.
- [ ] Error handling for invalid queries.
- [ ] Results are readable and sorted.

## Clean Code Bonus
- [ ] Modular files inside `src/`.
- [ ] Function names are clear.
- [ ] Short comments on complex logic.
- [ ] No dead code.

## Demonstration Readiness
- [ ] Prepare 2 examples for each query type.
- [ ] Keep one screenshot of UI.
- [ ] Keep one screenshot of saved index files.
- [ ] Prepare short explanation of AND/OR/NOT and proximity logic.

## Last Day Advice
- Freeze features early.
- Spend final time on testing, not adding new functionality.
- Keep backup copies of working index files and app.
