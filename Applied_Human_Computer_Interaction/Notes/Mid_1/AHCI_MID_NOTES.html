Shannon Entropy in HCI

Information Theory applied to Human-Computer Interaction

📊 What is Shannon Entropy?

Shannon Entropy is a measure of uncertainty or unpredictability in information. In HCI, it quantifies how predictable user choices are when interacting with interface elements like menus, buttons, or system responses.

Mathematical Definition

H = −Σ p(x) log₂ p(x)

Where p(x) is the probability of each possible event or user choice. The result is measured in bits.

Why Shannon Entropy Matters in HCI

  1. Measures uncertainty in user decisions — higher entropy = more uncertain choices
  2. Evaluates efficiency of menu designs — helps designers optimize layouts
  3. Predictive Interfaces — used to enhance or evaluate system predictions
  4. Interaction Speed — lower entropy = faster and easier interaction because choices are more predictable
🎯 Key Takeaway

Good interface design reduces entropy by making user choices predictable and clear. A well-designed menu with a dominant frequently-used option has lower entropy than one with equally confusing options.

🔢 Numerical Example – Complete Worked Solution

A software interface contains four menu options (A, B, C, D).

Case 1: Equal Probabilities (HIGH Uncertainty)

Option p(x) log₂ p(x) p(x) × log₂ p(x)
A 0.25 −2 −0.50
B 0.25 −2 −0.50
C 0.25 −2 −0.50
D 0.25 −2 −0.50
1 H = −[4 × (0.25 × log₂ 0.25)]
2 log₂ 0.25 = log₂ (1/4) = −2
3 H = −[4 × (0.25 × −2)] = −[4 × (−0.50)]
4 H = −(−2.0) = 2.0 bits
⚠️ Interpretation

H = 2 bits → Maximum entropy for 4 options. All options equally likely → user faces maximum uncertainty → slower decision-making and harder interaction.

Case 2: Unequal Probabilities (LOW Uncertainty)

Option p(x) log₂ p(x) p(x) × log₂ p(x)
A 0.70 −0.515 −0.3605
B 0.10 −3.322 −0.3322
C 0.10 −3.322 −0.3322
D 0.10 −3.322 −0.3322
1 H = −[0.70 × log₂(0.70) + 3 × (0.10 × log₂(0.10))]
2 log₂(0.70) ≈ −0.515  |  log₂(0.10) ≈ −3.322
3 H = −[(0.70 × −0.515) + 3 × (0.10 × −3.322)]
4 H = −[−0.3605 + 3 × (−0.3322)] = −[−0.3605 + (−0.9966)]
5 H = −(−1.3571) = ≈ 1.36 bits
✅ Interpretation

H ≈ 1.36 bits → Lower entropy. One dominant option (A at 70%) makes the interface more predictable → faster interaction and easier decision-making for users.

🎯 Exam Comparison Summary
Property Case 1 (Equal) Case 2 (Unequal)
Entropy 2.0 bits (HIGH) 1.36 bits (LOW)
Uncertainty Maximum Reduced
User Decision Speed Slower Faster
Design Quality Poor (confusing) Better (clear dominant)

Chapter 1 — The Human

Understanding human cognition, perception, memory, and reasoning for HCI design

🧠 HCI Definition & Core Concepts

HCI (Human-Computer Interaction) is the study of how people interact with computers and the design and evaluation of interactive systems.

Three Pillars of HCI

Pillar Focus Examples
Human Understanding user capabilities, limitations, cognition Memory limits, attention spans, motor abilities
Computer Technology, devices, I/O, display capabilities Keyboards, touchscreens, VR headsets
Interaction The communication between human and computer Clicks, gestures, voice commands, feedback

Why HCI Matters

  • Safety-critical systems — medical devices, air traffic control → poor design can cost lives
  • Productivity tools — efficiency depends on usability
  • Inclusion & accessibility — designing for all abilities

Key Insight: Technical vs. Practical

⚠️ Important

A system can be technically perfect but practically unusable if it doesn't align with human needs. Good HCI bridges this gap.

Cognitive Psychology in HCI

Essential for understanding how humans perceive, process, and store information. Key cognitive processes include: attention, perception, and problem-solving.

⚙️ Model Human Processor (MHP)

Proposed by Card, Moran, and Newell (1983). Treats humans as information processing systems with three interconnected subsystems:

Subsystem Function Analogy
Perceptual Processor Receives and buffers sensory input (visual, auditory, haptic) Computer's input devices (keyboard, mic)
Cognitive Processor Processes information, makes decisions, problem solves CPU / central processing unit
Motor Processor Executes physical responses and movements Output devices (printer, screen)
🎯 Exam Tip

MHP models human information processing as: Input → Processing → Output, just like a computer. Be ready to name all three subsystems and give examples of each.

👁️ Vision & Perception

Two Stages of Vision

  1. Physical Reception — light enters eye, hits retina (stimulus received)
  2. Processing & Interpretation — brain processes and makes sense of the visual data

Visual Capabilities & Limitations

Capability Description
Visual Acuity Ability to perceive fine detail — affects font size decisions
Contrast Sensitivity Ability to distinguish elements from background — critical for UI readability
Color Perception Cones = detail + color (bright light) | Rods = low light + movement detection

Visual Angle

Determines perceived size and depth. Familiar objects maintain "perceived size" despite distance changes (size constancy).

Reading Process

  • Saccades — rapid eye movements/fixations between points in text
  • Regressions — backward eye movements when meaning is unclear
  • Optimized by: proper font size, appropriate line length, good contrast

Optical Illusions

Demonstrate how the brain can misinterpret visual cues:

  • Müller-Lyer Illusion — lines with arrows appear different lengths
  • Ponzo Illusion — converging lines make identical objects seem different sizes
🎯 Exam Tip

Illusions prove that perception ≠ reality. This is why UI designers must test designs with real users — what the designer assumes may not match what users perceive.

🖐️ Hearing, Touch & Fitts's Law

Hearing (Auditory Perception)

  • Sound provides feedback (click sounds), notifications (alerts), and spatial cues (surround sound)
  • Important for accessibility — screen readers rely entirely on auditory output

Touch (Haptic Perception)

  • Crucial for users with visual impairments
  • Provides tactile feedback — vibrations, keyboard feel, force feedback
  • Used in modern devices: phone vibration, game controllers, VR gloves

Fitts's Law — Movement & Motor Control

MT = a + b × log₂(2D / W)

Where: MT = Movement Time | D = Distance to target | W = Width of target | a, b = constants

✅ Key Principles
  • Larger targets are faster to hit (bigger W → lower MT)
  • Closer targets are faster to reach (smaller D → lower MT)
  • UI implication: make frequently used buttons big and close to the user's likely cursor position

💾 Human Memory

Three Types of Memory

Type Duration Capacity Examples
Sensory Memory ~0.5–3 sec Large but brief Iconic (visual), Echoic (auditory)
Short-Term / Working Memory ~15–30 sec 7 ± 2 chunks Phone numbers, PINs, recent instructions
Long-Term Memory (LTM) Unlimited Vast Episodic (events) & Semantic (facts)

Long-Term Memory Structure

  • Episodic Memory — personal events and experiences (e.g., "I remember my first day at university")
  • Semantic Memory — factual knowledge (e.g., "Paris is the capital of France")
  • Organized as Semantic Networks — nodes (concepts) linked by associations; properties inherited from parent nodes

Memory Processes

Process Description HCI Implication
Storage Through rehearsal and repetition Consistent UI layouts help users memorize locations
Forgetting (Decay) Memory fades over time without use Don't rely on users remembering rarely used features
Forgetting (Interference) New info replaces/conflicts with old Avoid drastic UI changes that confuse existing users
Recognition Matching info to something seen before (EASIER) Menus, icons, visual cues — prefer over recall
Recall Retrieving from scratch without cues (HARDER) Command-line interfaces — harder for novices
🎯 Exam Tip

Recognition > Recall in UI design. Always prefer showing options (menus, icons) over requiring users to remember commands. The 7±2 rule means: never overload users with more than ~7 options at once.

🧩 Thinking, Reasoning & Problem Solving

Three Types of Reasoning

Type Direction Description Example
Deductive General → Specific Necessary conclusions from premises "All mammals breathe air. Dogs are mammals. → Dogs breathe air."
Inductive Specific → General Generalizing from observed cases "Every swan I've seen is white → all swans are white."
Abductive Effect → Cause Best explanation for an observation "Screen is black → probably the power is off."

Problem Solving Approaches

  • Insight (Gestalt) — sudden realization of the solution ("Aha!" moment)
  • Problem Space — searching through possible states/paths to reach a goal
  • Analogy — applying old knowledge/solutions to new problems

Skill Acquisition: Novice → Expert

Transition happens through Proceduralization — conscious rules become automatic, unconscious actions. Example: Learning to type — initially hunt-and-peck (deliberate), eventually touch-typing (automatic).

💡 Errors, Mental Models & Emotion

Types of Errors

Error Type Definition Example
Slips Correct plan, wrong execution Clicking "Delete" instead of "Download" (right intention, wrong button)
Mistakes Wrong plan, correct execution User formats drive thinking it frees up space, not realizing it erases data

Mental Models

A Mental Model is the user's internal theory of how a system works. If the mental model doesn't match the system model, errors occur. Good design aligns the system's behavior with user expectations.

Emotion (Affective Computing)

  • Positive affect → promotes creativity, exploration, and tolerance of minor issues
  • Negative affect (stress/anxiety) → narrows focus, can lead to frustration and errors

Individual Differences

Designers must account for: age, gender, physical ability, and cognitive styles to create inclusive designs accessible to everyone.

Chapter 2 — The Computer

Input/output devices, display technologies, VR, and processing in HCI

⌨️ Text Entry Devices

Interactivity Evolution

Era Type Characteristics
Past Batch Processing Punched cards, long waits, line printer output — if wrong, start over
Present Interactive Computing Rapid feedback, user is in control, focus on "doing rather than thinking"

Keyboards

  • QWERTY Layout — standard but NOT optimal for speed; designed for mechanical typewriters to prevent jamming
  • Persists due to technological and social inertia — users resist relearning, hardware replacement costly
  • Alternatives: DVORAK, alphabetic, chord keyboards

Chord Keyboards

  • Use very few keys (4-5); characters entered by pressing combinations (chords) of keys
  • Often reflect letter shapes (e.g., Microwriter)

Phone/Numeric Entry

  • Layouts differ: calculators vs. phones/ATMs — muscle memory plays a key role
  • T9 Multi-press: Press key multiple times (e.g., "Hello" = 4433555[pause]555666)
  • T9 Predictive: Type once per letter, dictionary guesses the word

Handwriting Recognition

Uses pen + digitizing tablet. Challenges: capturing stroke path/pressure, segmenting joined writing, interpreting styles, coping with individual differences.

Speech Recognition

  • Natural and familiar; accuracy >97% for limited vocabularies
  • Challenges: language ambiguity, accents, emotions, illness, background noise

Failed due to accidental activation. Feet are better for single-dimension controls (car pedals, sewing machines), not precise 2D pointing.

🖥️ Display Technologies

Bitmap Displays

Grid of pixels. Color depth: 1 bit (B&W), 8 bit (256 colors), 24-32 bit (True color / millions of colors).

Technology How It Works Pros Cons
CRT Electron gun hits phosphor screen Good color, fast refresh Bulky, radiation, heavy
LCD Voltage changes crystal polarization Thin, light, no radiation, less eye strain Viewing angle limits
Gas Plasma Gas ionization Large screens, high brightness Power hungry
DVST Direct View Storage Tube Persistent display, no flicker Cannot selectively erase

Situated Displays

  • Displays in public places (info boards) or specific locations (door displays)
  • Meaning is location-dependent
  • Hermes Example: Small door displays for leaving handwritten notes, readable via web

Printing & Typography

  • DPI (Dots per inch): Laser printers: 600-2400 dpi
  • Serif fonts (Times) → better for long printed text
  • Sans-serif fonts (Arial/Helvetica) → better for screens
  • Lowercase → easier to read (word shape recognition)
  • Uppercase → better for individual codes (BA793)

🥽 VR, 3D & Advanced Interaction

3D Positioning

Moves from 2DOF (X, Y) to 6DOF (X, Y, Z + Pitch, Yaw, Roll). Devices: SpaceMouse (CAD), Meta Quest controllers.

VR Technology

  • Data Gloves — fiber optics track gestures, provide haptic feedback
  • VR Headsets — use foveated rendering (GPU power focused where you look)
  • Motion Sickness — caused by time delay (lag) and depth perception conflicts (eye angle vs. focus plane)

📡 Output, Sensing & Processing

Visual Status Indicators

  • LEDs — quick status feedback (charging, Wi-Fi)
  • Dials/Gauges — analog visuals faster for humans to spot changes/extremes
  • HUD (Head-Up Display) — shows info in line of sight (fighter jets), keeps attention on environment

Haptic Output

Vibration and force feedback (surgical simulations, BMW iDrive menu "bumps").

Environment & Bio-Sensing

Type Examples
Environmental Infrared (auto washbasins), GPS, movement, pressure sensors
Behavioral Activity monitoring, vocabulary changes, typing rate
Physiological Heart rate, stress (smartwatches), face unlock (biometrics)

Memory & Processing in Computers

  • RAM — short-term, volatile, very fast (~100 ns)
  • LTM (Storage) — persistent (SSD/HDD/Cloud), essential for auto-save and version history

Processing Bottlenecks

Problem Effect on User
Slow Processor "Icon Wars" — windows flash suddenly after multiple impatient clicks
Too-Fast Processor Helpful text or stories scroll too quickly to read
Computation Bound CPU maxed out → system freezes
Storage Bound Disk I/O bottleneck → slow file operations
Graphics Bound GPU bottleneck → lag in rendering
Network Bound Buffering, slow loading — most common modern bottleneck

🔄 User-Computer Communication

Why Interaction Matters

  • Users must communicate requirements to the system
  • The computer must respond appropriately

Communication Channels

Channel Direction Examples
Input User → System Clicks, taps, typing, voice commands
Output System → User Screens, sound, vibration, animations

Levels of Interaction

Type Description Example

Interactive (High) Continuous instructions + instant feedback Dragging files, pinch-to-zoom, VR games

Key Interaction Components

  • Interaction Models — conceptual frameworks explaining how users & systems interact
  • Ergonomics — designing systems that fit human physical abilities (one-handed phone use, dark mode)
  • Interaction Styles — nature of dialog (simple tap vs. complex multi-step workflows)
  • Context — Social (group chats) or Organizational (university LMS)

Key Terms

Term Definition Example
Domain Area of work under study Graphic design

🔁 Norman's Model of Interaction

One of the most influential models in HCI. Describes interaction as a cycle between user and computer: the Execution/Evaluation Loop.

The 7 Stages of the Execution/Evaluation Loop

1 Establish Goal — User decides what they want to achieve
2 Form Intention — User formulates a plan of action
3 Specify Action — User identifies specific interface actions needed
4 Execute Action — User physically performs the action
5 Perceive State — User observes the system's response
6 Interpret State — User makes sense of what happened
7 Evaluate State — User compares result against original goal
Phase Stages Description
Execution Phase Stages 1–4 Goal → Intention → Action Specification → Execution
Evaluation Phase Stages 5–7 Perception → Interpretation → Evaluation
✅ Complete Example: Switching on a Light

Goal: Get more light (evening falls while reading)

Intention: Switch on the desk lamp

Action Specification: Reach for lamp, find switch

Execution: Press the switch

Perception: See light come on (or not)

Interpretation: Understand the lamp is now on

Evaluation: Is there enough light? → Goal achieved or repeat

⚠️ Key Insight: Multiple Paths

Same goal can have different intentions. For "get more light": switch desk lamp, turn on ceiling light, ask someone, or open curtains. Goal ≠ Intention ≠ Action.

🔀 Gulf of Execution & Gulf of Evaluation

Gulf of Execution

"How do I do what I want to do?"

The gap between what the user wants to do and what actions the system allows.

✅ Example

Problem: Switch is hidden, oddly shaped, or unclear. User doesn't know which switch controls the lamp.

Solution: Design visible, labeled, and easy-to-press switches → reduce the gulf.

HCI Principle: Make actions discoverable, visible, and intuitive.

Gulf of Evaluation

"Did it work? What happened?"

The gap between what the system shows and what the user expects or understands.

✅ Example

Problem: User presses switch but light doesn't turn on. No indication why (blown bulb? no power?).

Solution: Provide immediate feedback (light turns on) or a clear indicator showing what went wrong.

HCI Principle: Always give clear, timely feedback about system state.

🎯 Exam Summary: Two Gulfs
Gulf Question Problem Solution
Execution "How do I do this?" Interface actions are unclear/hidden Make controls visible, labeled, intuitive
Evaluation "Did it work?" System feedback is missing/confusing Provide immediate, clear feedback

Q1 Shannon Entropy

This question will test your understanding of Shannon Entropy theory and your ability to solve numerical problems.

What You Must Know

  • Definition: Shannon Entropy measures uncertainty/unpredictability in information
  • Formula: H = −Σ p(x) log₂ p(x)
  • Units: Measured in bits
  • HCI Application: Quantifies predictability of user choices in interface elements

Step-by-Step Solving Method

1 Identify all options and their probabilities p(x). Verify they sum to 1.0
2 Calculate log₂ of each probability. Key values: log₂(0.25) = −2, log₂(0.5) = −1, log₂(0.10) ≈ −3.322, log₂(0.70) ≈ −0.515
3 Multiply each p(x) × log₂ p(x)
4 Sum all products
5 Apply the negative sign: H = −(sum)
6 Interpret: Higher H = more uncertainty = worse design | Lower H = more predictable = better design

Quick Reference: Common log₂ Values

p(x) log₂ p(x) p(x) × log₂ p(x)
1.0 0 0
0.5 −1 −0.5
0.25 −2 −0.5
0.125 −3 −0.375
0.10 −3.322 −0.3322
0.70 −0.515 −0.3605
0.50 −1.0 −0.50
0.30 −1.737 −0.5211
0.20 −2.322 −0.4644
🎯 Remember

Maximum entropy for N equally likely options = log₂(N). So for 4 options: max H = 2 bits. For 8 options: max H = 3 bits. If your answer exceeds this, you made a calculation error!

Q2 Doner Model (Norman's Model of Interaction)

This question tests your understanding of Norman's Execution/Evaluation Loop and the Gulfs.

Complete Answer Framework

Part A: The 7 Stages

# Stage Phase Description
1 Establish Goal EXECUTION User decides what they want
2 Form Intention User plans how to achieve it
3 Specify Action User identifies interface actions
4 Execute Action User physically performs action
5 Perceive State EVALUATION User observes system response

7 Evaluate State User compares result vs. goal

Part B: The Two Gulfs

Gulf Meaning User's Question Design Solution
Gulf of Execution Gap between user intent and available actions "How do I do this?" Visible, labeled, intuitive controls
Gulf of Evaluation Gap between system state and user understanding "Did it work?" Immediate, clear feedback

Part C: How to Write a Scenario Answer

✅ Example: Ordering Food on a Delivery App
  1. Goal: Order dinner
  2. Intention: Open food delivery app and find a restaurant
  3. Action Spec: Tap app icon → Search "pizza" → Select restaurant
  4. Execute: Tap "Add to cart" → Tap "Checkout" → Confirm payment
  5. Perceive: See "Order Confirmed" screen with estimated delivery time
  6. Interpret: Understand order is placed and food is being prepared
  7. Evaluate: Correct items? Correct address? → Goal achieved

Gulf of Execution: If the checkout button is hidden or confusing → user can't complete order

Gulf of Evaluation: If no confirmation screen → user doesn't know if order went through
